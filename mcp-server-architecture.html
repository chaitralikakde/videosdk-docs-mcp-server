Recursive File Walker

loadAllDocs(dir) walks ai_agents/ recursively, reads every .md file, and resolves relative paths via __dirname.

  • Reads directory entries
  • Recurses into subdirectories
  • Reads .md file content
  • Passes to chunker
✂️

Markdown Chunker

chunkByHeadings() splits at #, ##, ### headings. Each chunk = { filePath, heading, text }.

  • Scan lines for heading patterns
  • Flush accumulated text as a chunk
  • Start new chunk at each heading
  • Return array of structured chunks
🧠

In-Memory Store

ALL_CHUNKS[] — a flat array holding all 1,343 chunks in memory. Loaded once at startup, queried on every search call.

  • 120 files → 1,343 chunks
  • Each chunk: path + heading + text
  • No database, no files, no embeddings
② Search
Engine
🔤

Tokenizer

tokenize(str) extracts lowercase alphanumeric words via regex /[a-z0-9]+/g. Strips punctuation, normalizes case.

  • "OpenAI TTS plugin" → ["openai", "tts", "plugin"]
  • Applied to query, headings, and body text
⚖️

Weighted Scorer

scoreChunk() scores every chunk against query tokens with a weighting system.

  • Exact match in heading → ×3
  • Exact match in body → ×1
  • Substring match in heading → ×1.5
  • Substring match in body → ×0.5
🏆

Ranking & Top-K

searchDocs(query, topK) scores all 1,343 chunks, filters out zeros, sorts descending, returns top 5.

  • Score all chunks
  • Filter score > 0
  • Sort by score descending
  • Slice top 5 results
③ MCP
Tools
🔍

SearchDocsGoogle

Takes a query string. Returns top 5 matching chunks with file path, section heading, score, and 1500-char preview.

INPUT: { query: "openai tts" }

OUTPUT: Ranked results with context

📋

list_docs

No arguments. Returns all 120 unique file paths — a complete table of contents of the documentation tree.

INPUT: {}

OUTPUT: Sorted file path list

📄

get_doc

Takes a relative path (from list_docs). Returns the full raw markdown content of that file.

INPUT: { path: "plugins/llm/openai.md" }

OUTPUT: Full markdown content

④ Transport
Layer
🔌

StdioServerTransport

Communicates via stdin / stdout using JSON-RPC. Claude Code launches the server as a subprocess and sends/receives MCP protocol messages over the pipe.

  • Claude Code spawns node server.js
  • Sends JSON-RPC over stdin
  • Receives tool results over stdout
  • stderr used for logging only
🤖

Client (Claude Code / Desktop)

The AI calls ListTools to discover the 3 tools, then CallTool to search, list, or read docs — all transparently during conversation.

  • Discovers tools at session start
  • Calls SearchDocsGoogle for queries
  • Calls list_docs to browse
  • Calls get_doc to read full files
💬

Typical Flow

User asks about VideoSDK → Claude searches docs → reads relevant file → gives grounded answer with code examples.

  • User: "How to use Deepgram STT?"
  • Claude → SearchDocsGoogle("deepgram stt")
  • Claude → get_doc("plugins/stt/deepgram.md")
  • Claude answers with full context
